# 智能体检报告解读助手 - PRD v1.0

> 状态：草稿 | 作者：PM求职作品集 | 日期：2026-05-19

## 一、产品概述

### 一句话定位
为普通体检用户提供**即时、通俗、可信**的AI体检报告解读，帮他们回答三个核心问题：
- 「这个指标高了/低了严重吗？」
- 「我到底有没有病？」
- 「我该不该去医院？看哪个科？」

### 目标用户

| 用户画像 | 典型场景 | 核心诉求 | 频次 |
|----------|----------|----------|------|
| **小白型**（28岁互联网从业者） | 公司年度体检，拿到报告一脸懵，十几个箭头不知道看哪个 | 「有没有大问题？别吓我」 | 1次/年 |
| **焦虑型**（35岁新手妈妈） | 体检多个指标异常，百度越查越怕，不想随便挂号 | 「我到底严不严重？该挂什么科？」 | 1-3次/年 |
| **孝心型**（32岁子女） | 爸妈体检报告拿来，自己看不懂还要解释给父母听 | 「帮我翻译成白话，告诉我要不要催我妈去看医生」 | 2次/年（父母各一份） |
| **管理型**（45岁慢病前期） | 连续几年体检，想对比趋势，不想每次都去找医生看报告 | 「跟去年比是好是坏？生活方式需要调整吗？」 | 1次/年后+日常 |

### 核心假设（MVP待验证）

1. 用户最痛的不是「不知道指标什么意思」，而是「不知道严重不严重、该不该去医院」
2. 如果AI解读能让用户判断「这个不用去医院」vs「这个建议挂XX科看一下」，用户会持续使用
3. 用户愿意上传体检报告图片（习惯已在微信/支付宝/京东健康中培养）

## 二、用户旅程（Happy Path）

```
① 用户拿到体检报告 → 拍照/上传PDF
       │
       ▼
② 系统OCR识别 → 提取指标名+数值+单位+参考范围
       │
       ▼
③ AI分析（RAG增强）：
   - 逐个指标判断：正常/偏高/偏低/需关注
   - 交叉关联分析：哪些异常是同一问题的不同表现？
   - 紧急度分层：🟢正常 🟡观察 🔴建议就医 🔴需急诊
       │
       ▼
④ 生成结构化解读报告：
   - 总结：总体情况一句话
   - 异常项：每个异常指标的通俗解释 + 风险等级
   - 就医建议：挂什么科、做什么进一步检查
   - 生活建议：饮食/运动/复查周期
       │
       ▼
⑤ 用户可追问（对话模式）：
   - 「这个指标跟我上次比是好是坏？」
   - 「我长期吃XX药，这个异常跟它有关吗？」
   - 「如果不去医院，我需要观察什么？」
```

## 三、MVP功能范围

### V1.0（做）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 体检报告图片/PDF上传 | P0 | 支持拍照、相册、文件上传 |
| OCR提取指标 | P0 | 识别指标名称、数值、单位、参考范围 |
| AI单指标解读 | P0 | 通俗解释每个异常指标 |
| 综合风险评估 | P0 | 多指标交叉分析+紧急度分层 |
| 就医科室建议 | P0 | 建议挂什么科、做什么检查 |
| 对话追问 | P1 | 用户可以针对结果继续提问 |
| 历史对比 | P1 | 支持上传历史报告，对比趋势 |

### V1.0（砍——产品经理的能力）

| 功能 | 优先级 | 为什么砍 |
|------|--------|----------|
| 直接连接可穿戴设备数据 | P2 | 技术复杂度高，用户基数小，MVP阶段不值得 |
| 医生在线1v1 | P2 | 双边市场冷启动问题，不是AI产品MVP该做的 |
| 药品推荐/商城 | 永远不做 | 医疗伦理红线，体检→药品导流是违规的 |
| AI诊断输出 | 永远不做 | 我们做「解读和就医建议」，不是「诊断」，法律/伦理边界 |
| 匿名社区/分享 | P2 | 健康数据隐私极度敏感，MVP不碰社交 |

## 四、AI方案设计

### 为什么选LLM+RAG而非纯LLM？
（详见竞品分析1.3节——通用LLM做不到精确、可溯源、可更新的医疗知识引用）

### 技术架构（简化版）
```
前端（React）→ 上传报告
    │
    ▼
后端（FastAPI）
    ├── OCR服务（提取文本）
    ├── 预处理（格式化指标数据）
    ├── RAG检索（查询医疗知识库）
    └── LLM调用（生成解读）
    │
    ▼
返回结构化解读 → 前端展示
```

### AI评估指标（产品视角）

| 指标 | 定义 | 目标（MVP） | 备注 |
|------|------|------------|------|
| 指标识别准确率 | OCR+解析后正确识别的指标数/总指标数 | >90% | 手动验证20份真实报告 |
| 严重度分层准确率 | AI判断的紧急度与临床常识一致的比例 | >85% | 用你的临床知识做金标准 |
| 用户理解度 | 用户读完解读后能否正确回答「我该不该去医院」 | >90%（用户测试） | 核心指标 |
| 用户信任度 | 用户对AI解读的信任评分（1-5分） | >3.5 | 用户测试后问卷 |

## 五、风险与缓解

| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| AI给出错误医学建议导致用户延误就医 | 🔴 致命 | ①每个解读都附带「AI辅助解读，不构成医疗建议」免责；②明确标注紧急度而非确定性诊断；③P2+异常指标强制建议就医 |
| 用户隐私（体检报告是最敏感的隐私数据之一） | 🔴 高 | ①不上传服务器做永久存储（处理后删除原图）；②本地OCR优先；③明确隐私政策 |
| 幻觉——编一个不存在的指标或参考范围 | 🔴 高 | RAG+知识库约束；结构化输出校验（必须引用知识库来源） |
| OCR对中文手写/印章的识别率低 | 🟡 中 | V1只支持打印版报告；异常格式做兜底（手动输入） |
